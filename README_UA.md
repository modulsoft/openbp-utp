# HTTP-сервіс OpenBP для 1C:УТП

[🇬🇧 English version](README.md)

## Огляд

Цей репозиторій призначений для плавного переходу від системи обліку 1С та конфігурації УТП до платформи OpenBP. Після встановлення відповідного сервісу, 1С можна буде автоматично синхронізуватися з OpenBP, забезпечуючи безперебійний обмін даними та поступову міграцію.

HTTP-сервіс OpenBP надає RESTful API, який представляє документи та довідники 1C:УТП у вигляді JSON-ендпоінтів. Це дозволяє зовнішнім системам (включаючи OpenBP):
- Запитувати та отримувати дані довідників (номенклатура, організації, контрагенти)
- Створювати та оновлювати документи (прихідні накладні, надходження)
- Синхронізувати довідники та транзакції в режимі реального часу

## Можливості

- ✅ RESTful API з форматом запитів/відповідей JSON
- ✅ Підтримка CORS для інтеграції веб-додатків
- ✅ Автентифікація на основі UUID користувача
- ✅ Підтримка документів надходження товарів (ПоступлениеТоваровУслуг)
- ✅ Операції з довідниками номенклатури та організаційних даних
- ✅ Автоматичне проведення та валідація документів

## Передумови

Перед встановленням цього HTTP-сервісу переконайтеся, що у вас є:

1. **Платформа 1C:Підприємство** версії 8.3 або вище
2. **Конфігурація 1C:УТП (Управління торговим підприємством)**
3. **Необхідні модулі/підсистеми 1C**:
   - `ОбработкаJSON` - модуль обробки JSON
   - `УправлениеПользователями` - модуль управління користувачами
   - `БухгалтерскийУчет` - модуль бухгалтерського обліку
   - `УправлениеВзаиморасчетами` - управління взаєморозрахунками
4. **Права адміністратора** конфігурації 1C
5. **Можливість публікації HTTP-сервісів** увімкнена в інформаційній базі

## Встановлення

### Крок 1: Створення HTTP-сервісу

1. Відкрийте конфігурацію 1C:УТП у **режимі Конфігуратора**
2. У Конфігураторі перейдіть до: **Загальні → HTTP-сервіси** (Общие → HTTP-сервисы)
3. Створіть новий HTTP-сервіс з наступними властивостями:
   - **Ім'я**: `openbp`
   - **Кореневий URL**: `openbp` (або бажаний вами шлях)
   - **Опис**: "Сервіс інтеграції OpenBP"
4. Скопіюйте весь вміст із файлу [`openbp_HTTP.bsl`](openbp_HTTP.bsl)
5. Вставте його в редактор модуля HTTP-сервісу
6. Додайте **Шаблони URL** для кожної операції:

   | Шаблон URL | HTTP-метод | Функція-обробник |
   |------------|------------|------------------|
   | `/documents/{uuid}` | POST | `ДокументыИзменитьДанные` |
   | `/documents/{uuid}` | OPTIONS | `Options` |
   | `/catalogs/{uuid}` | GET | `СправочникиПолучитьДанные` |
   | `/catalogs/{uuid}` | POST | `СправочникиИзменитьДанные` |
   | `/catalogs/{uuid}` | OPTIONS | `Options` |
   | `/head` | ANY | `head` |

4. Збережіть конфігурацію HTTP-сервісу

### Крок 2: Оновлення та публікація конфігурації

1. **Оновіть конфігурацію бази даних**: Конфігурація → Оновити конфігурацію бази даних
2. Вирішіть будь-які конфлікти або помилки, якщо вони виникнуть
3. Перезапустіть 1C:Підприємство у **режимі Конфігуратора від імені Адміністратора** 
4. **Опублікуйте HTTP-сервіс**:
   - Відкрийте: Адміністрування → Публікація на веб-сервері
   - Увімкніть публікацію для сервісу `openbp`
   - Зверніть увагу на опублікований URL (зазвичай: `http://your-server:port/your-base/hs/openbp/`)

### Крок 3: Налаштування доступу користувачів

1. Створіть або визначте обліковий запис користувача для доступу до API
2. Зверніть увагу на **UUID** користувача (функція **УникальныйИдентификатор()** повертає значення uuid для об'єкта записаного в базі)
3. Переконайтеся, що користувач має відповідні дозволи:
   - Доступ на читання/запис до довідників: Контрагенти, Номенклатура, Організації, Склади
   - Доступ на читання/запис до документів: ПоступлениеТоваровУслуг
   - Дозвіл на проведення документів
